# Video 3 (Routing)
Halaman login -> kenapa bisa muncul halaman login?
1. halaman login bisa muncul karena ada karakter '/' (route utama http://127.0.0.1:8000/) di web.php yang mengembalikan view 'auth.login' (file login di folder auth).
2. karna kita akan membuat aplikasi manajemen pegawai, kita buat route '/pegawai' di web.php. Setelah membuat route, kita bisa buatkan sebuah view untuk route tersebut menggunakan perintah php artisan make:view.
3. perintah familiar berikutnya adalah 'redirect'. mengarahkan route yang kita buat ke route yang sudah ada atau website yang berbeda 'return redirect()-> away(https://youtube.com);'
4. Baca Dokumentasi laravel.
5. kemudian untuk penamaan route, digunakan agar saat kita melakukan refirect, kita tidak perlu menuliskan route nya secara spesifik, kita hanya perlu membuat nama dari route tersebut lalu memganggilnya pada perinteah redirect. Hal ini juga memiliki keuntungan di mana saat kita mengganti route asli, redirect yang ada tidak akan error, karna yang kita ambil namanya, bukan route nya.
6. fallback routes akan dieksekusi ketika laravel tidak menemukan route yang kita akses, hasilnya pasti akan 404.

# Video 4 (Model dan Migrasi)
Migrasi adalah cara laravel untuk mengatur struktur table database lewat code (tidak seperti pma).
Model adalah representasi dari table yang kita miliki di database. Model juga akan kita gunakan untuk melakukan kkueri ke database dan juga membuat relasi antar table.
Untuk menambahkan kolom saat data sudah banyak ke dalam table, adalah dengan menggunakan perintah seperti ini:
'php artisan make:migration add_nip_to_table_pegawais --table=pegawais'

Untuk rename suatu kolom pada table di database dapat menggunakan perintah:
'php artisan make:migration rename_nip_to_nik_table_pegawais --table=pegawais'

# Video 5 (Query, Factory, Seeder)
Seeder bisa membantu kita mengisi data ke dalam database secara otomatis. Contoh dari perintah seed adalah:
'php artisan make:seeder UserSeeder'
Namun sebelum melakukan seed data, kita harus pergi ke model untuk melakukan konfigurasi field mana saja yang bisa diisi,
biasanya dilakukan dengan 'protected $guarded = ['id']'
artinya, id tidak bisa diisi manual oleh user karena diisi otomatis oleh sistem.
kemudian ada satu library yang dinamakan Faker, library ini menyediakan data data dummy.
pada Faker kita bisa mengatur sebanyak apa data dan bahasa apa yang ingin digunakan (selama bahasa available pada library)
kita juga bisa menggunakan factory untuk mengisi data di dalam table. pada factory kita tidak perlu melakukan import faker, kita akan menggunakan $this->faker.
namun pada factory, defaultnya inggris, ganti saja faker locale pada .env jika diperlukan.
pada seeder, kita ganti kalau sebelumnya menggunakan faker, jadi menggunakan factory create. pada model juga harus kita beri tahu dengan membuat 'use HasFactory'.

- Eloquent
eloquent adalah cara melakukan queri yang dipermudah oleh laravel, misalnya saat kita ingin menampilkan semua data dari suatu table, biasanya kita akan menggunakan 'SELECT * FROM TABLE' namun engan eloquent kita bisa menggunakan namaModel::all(), dan lainnya
